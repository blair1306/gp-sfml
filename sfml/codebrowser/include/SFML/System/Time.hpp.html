445/// \class sf::Time
446/// \ingroup system
447///
448/// sf::Time encapsulates a time value in a flexible way.
449/// It allows to define a time value either as a number of
450/// seconds, milliseconds or microseconds. It also works the
451/// other way round: you can read a time value as either
452/// a number of seconds, milliseconds or microseconds.
453///
454/// By using such a flexible interface, the API doesn't
455/// impose any fixed type or resolution for time values,
456/// and let the user choose its own favorite representation.
457///
458/// Time values support the usual mathematical operations:
459/// you can add or subtract two times, multiply or divide
460/// a time by a number, compare two times, etc.
461///
462/// Since they represent a time span and not an absolute time
463/// value, times can also be negative.
464///
465/// Usage example:
466/// \code
467/// sf::Time t1 = sf::seconds(0.1f);
468/// Int32 milli = t1.asMilliseconds(); // 100
469///
470/// sf::Time t2 = sf::milliseconds(30);
471/// Int64 micro = t2.asMicroseconds(); // 30000
472///
473/// sf::Time t3 = sf::microseconds(-800000);
474/// float sec = t3.asSeconds(); // -0.8
475/// \endcode
476///
477/// \code
478/// void update(sf::Time elapsed)
479/// {
480/// position += speed * elapsed.asSeconds();
481/// }
482///
483/// update(sf::milliseconds(100));
484/// \endcode
485///
486/// \see sf::Clock
487///
