287////////////////////////////////////////////////////////////
288/// \class sf::SoundBuffer
289/// \ingroup audio
290///
291/// A sound buffer holds the data of a sound, which is
292/// an array of audio samples. A sample is a 16 bits signed integer
293/// that defines the amplitude of the sound at a given time.
294/// The sound is then reconstituted by playing these samples at
295/// a high rate (for example, 44100 samples per second is the
296/// standard rate used for playing CDs). In short, audio samples
297/// are like texture pixels, and a sf::SoundBuffer is similar to
298/// a sf::Texture.
299///
300/// A sound buffer can be loaded from a file (see loadFromFile()
301/// for the complete list of supported formats), from memory, from
302/// a custom stream (see sf::InputStream) or directly from an array
303/// of samples. It can also be saved back to a file.
304///
305/// Sound buffers alone are not very useful: they hold the audio data
306/// but cannot be played. To do so, you need to use the sf::Sound class,
307/// which provides functions to play/pause/stop the sound as well as
308/// changing the way it is outputted (volume, pitch, 3D position, ...).
309/// This separation allows more flexibility and better performances:
310/// indeed a sf::SoundBuffer is a heavy resource, and any operation on it
311/// is slow (often too slow for real-time applications). On the other
312/// side, a sf::Sound is a lightweight object, which can use the audio data
313/// of a sound buffer and change the way it is played without actually
314/// modifying that data. Note that it is also possible to bind
315/// several sf::Sound instances to the same sf::SoundBuffer.
316///
317/// It is important to note that the sf::Sound instance doesn't
318/// copy the buffer that it uses, it only keeps a reference to it.
319/// Thus, a sf::SoundBuffer must not be destructed while it is
320/// used by a sf::Sound (i.e. never write a function that
321/// uses a local sf::SoundBuffer instance for loading a sound).
322///
323/// Usage example:
324/// \code
325/// // Declare a new sound buffer
326/// sf::SoundBuffer buffer;
327///
328/// // Load it from a file
329/// if (!buffer.loadFromFile("sound.wav"))
330/// {
331/// // error...
332/// }
333///
334/// // Create a sound source and bind it to the buffer
335/// sf::Sound sound1;
336/// sound1.setBuffer(buffer);
337///
338/// // Play the sound
339/// sound1.play();
340///
341/// // Create another sound source bound to the same buffer
342/// sf::Sound sound2;
343/// sound2.setBuffer(buffer);
344///
345/// // Play it with a higher pitch -- the first sound remains unchanged
346/// sound2.setPitch(2);
347/// sound2.play();
348/// \endcode
349///
350/// \see sf::Sound, sf::SoundBufferRecorder
351///
