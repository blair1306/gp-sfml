189/// \class sf::Keyboard
190/// \ingroup window
191///
192/// sf::Keyboard provides an interface to the state of the
193/// keyboard. It only contains static functions (a single
194/// keyboard is assumed), so it's not meant to be instantiated.
195///
196/// This class allows users to query the keyboard state at any
197/// time and directly, without having to deal with a window and
198/// its events. Compared to the KeyPressed and KeyReleased events,
199/// sf::Keyboard can retrieve the state of a key at any time
200/// (you don't need to store and update a boolean on your side
201/// in order to know if a key is pressed or released), and you
202/// always get the real state of the keyboard, even if keys are
203/// pressed or released when your window is out of focus and no
204/// event is triggered.
205///
206/// Usage example:
207/// \code
208/// if (sf::Keyboard::isKeyPressed(sf::Keyboard::Left))
209/// {
210/// // move left...
211/// }
212/// else if (sf::Keyboard::isKeyPressed(sf::Keyboard::Right))
213/// {
214/// // move right...
215/// }
216/// else if (sf::Keyboard::isKeyPressed(sf::Keyboard::Escape))
217/// {
218/// // quit...
219/// }
220/// \endcode
221///
222/// \see sf::Joystick, sf::Mouse, sf::Touch
223///
